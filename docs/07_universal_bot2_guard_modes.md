# Universal Bot#2 Guard Modes — аналитик, критик и креативный компонент

## Цель

Bot#2 должен быть не только Code Guard. Он должен работать как универсальный второй слой контроля и усиления качества:

- аналитик;
- критик;
- креативный challenger;
- risk gate;
- quality gate;
- human escalation gate.

Главная идея: Bot#1 делает работу, Bot#2 проверяет риски, усиливает слабые места и не даёт принять формальный или ошибочный результат.

## Базовые роли

### Bot#1

Исполнитель. Обычно DeepSeek.

Задачи:

- быстро сделать черновик;
- написать код;
- собрать таблицу;
- извлечь данные;
- подготовить письмо;
- предложить идею;
- сделать первый расчёт;
- собрать первичную аналитику.

### Bot#2

Проверяющий и усилитель. Обычно OpenAI/Codex.

Задачи:

- найти ошибку;
- найти слабую логику;
- проверить факты;
- проверить расчёты;
- проверить источники;
- предложить альтернативу;
- усилить идею;
- поймать формальное выполнение;
- спросить пользователя, если решение рискованное.

## Режимы Bot#2

### 1. Quality Gate

Используется для кода, документов, КП, таблиц, тендеров и любых результатов, которые могут быть приняты как готовые.

Проверяет:

- выполнено ли исходное ТЗ;
- нет ли пропущенных пунктов;
- нет ли формального выполнения;
- есть ли доказательства;
- нужно ли вернуть Bot#1 на исправление.

### 2. Analytical Validator

Используется для аналитики, расчётов, тендеров, поиска, RAG и бизнес-выводов.

Проверяет:

- факты;
- источники;
- свежесть данных;
- допущения;
- формулы;
- альтернативные объяснения;
- противоречия;
- уровень уверенности.

Bot#2 не должен просто соглашаться с Bot#1. Он обязан искать слабые места.

### 3. Creative Challenger

Используется для идей, стратегий, позиционирования, коммерческих предложений, переговоров, упаковки продукта и новых сервисов.

Проверяет и усиливает:

- оригинальность идеи;
- практичность;
- соответствие бизнесу пользователя;
- слабые места;
- скрытые риски;
- альтернативные варианты;
- более сильную формулировку;
- более дешёвую или быструю реализацию.

Creative Challenger не должен тормозить каждую идею. Его задача — улучшить идею или показать, почему она слабая.

## Где Bot#2 обязателен как аналитик

- тендерная классификация;
- ГОЗ, 223-ФЗ, 275-ФЗ, нацрежим;
- расчёт цены, НДС, маржи, дивидендов;
- КП клиенту;
- сравнение поставщиков;
- RAG-ответ по базе знаний;
- запись нового правила в память;
- импорт Excel/PDF/OCR;
- автоматическое решение по заявке;
- вывод, который влияет на деньги, клиента, поставщика, тендер или производство.

## Где Bot#2 обязателен как креативный challenger

- стратегия продукта;
- новый сервис;
- новая бизнес-гипотеза;
- коммерческое предложение с высокой ценой;
- предложение партнёрства;
- важное письмо клиенту или поставщику;
- упаковка проекта для заказчика;
- презентация или pitch;
- изменение позиционирования компании;
- идея, на которую будет потрачено много времени или денег.

## Где Bot#2 лишний

- простая команда `/status`;
- короткий перевод;
- простое объяснение термина;
- черновой brainstorm без принятия решения;
- уведомление в Telegram;
- пересказ короткого сообщения;
- внутренняя заметка без влияния на память проекта;
- документ, который не будет отправляться наружу и не меняет правила.

## Light-mode

Bot#2 можно включать в light-mode, если задача средняя по риску.

Light-mode проверяет:

- не потерян ли смысл;
- нет ли очевидной ошибки;
- не забыты ли важные ограничения;
- нет ли риска для клиента, денег или памяти;
- нужно ли включить full review.

## Full-mode

Full-mode нужен, если ошибка дорогая.

Full-mode проверяет:

- факты;
- формулы;
- источники;
- бизнес-риски;
- юридические/финансовые риски;
- противоречия;
- качество результата;
- альтернативы;
- необходимость решения пользователя.

## Human escalation

Bot#2 обязан спросить пользователя, если:

- Bot#1 и Bot#2 не согласны после 3 раундов;
- решение влияет на деньги;
- решение влияет на клиента или поставщика;
- решение меняет правила проекта;
- нужно записать новое правило в память;
- есть юридический или тендерный риск;
- креативная идея требует существенных вложений;
- аналитика имеет низкую уверенность.

## Формат Telegram-сообщения

```text
Сообщение от Bot#2

Режим: Analytical Validator / Creative Challenger / Quality Gate

Кратко: ...

Что сделал Bot#1:
- ...

Что нашёл Bot#2:
- ...

Риск:
- ...

Рекомендация Bot#2:
- ...

Да — согласен с Bot#2, вернуть Bot#1 на доработку.
Нет — принять вариант Bot#1.
```

## Итоговая формула

Bot#1 делает.

Bot#2 проверяет, усиливает и спорит.

Пользователь решает только там, где есть цена ошибки.
